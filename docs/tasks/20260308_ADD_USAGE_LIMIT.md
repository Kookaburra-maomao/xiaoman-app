# 添加使用次数限制功能

## 任务时间
2026-03-08

## 功能概述
为了控制大模型调用成本，添加每日使用次数限制功能。在用户进行对话、上传图片、生成日记操作前，先检查当日使用次数，超过限制则提示用户。

## 使用限制配置

| 功能类型 | 每日限制 | 说明 |
|---------|---------|------|
| 对话 (chat) | 100次 | 文字对话次数 |
| 上传图片 (image) | 5次 | 上传图片并理解次数 |
| 生成日记 (diary) | 3次 | 生成日记次数 |

## 服务端接口

### 接口信息
- **路径**: `GET /api/chat/today-count`
- **功能**: 查询指定用户当天的对话数量
- **支持**: 按类型统计（chat、image、diary）

### 请求参数
- `userId` (必填): 用户ID
- `type` (可选): 对话类型（chat、image、diary）

### 响应格式

#### 查询所有类型（不传 type）
```json
{
  "code": 200,
  "message": "查询成功",
  "data": {
    "userId": "1234567890123456",
    "date": "2026-03-08",
    "total": 25,
    "byType": {
      "chat": 15,
      "image": 8,
      "diary": 2
    }
  }
}
```

#### 查询指定类型（传 type）
```json
{
  "code": 200,
  "message": "查询成功",
  "data": {
    "userId": "1234567890123456",
    "date": "2026-03-08",
    "type": "chat",
    "count": 15
  }
}
```

## 客户端实现

### 1. 新增服务文件 ✅
**文件**: `services/usageLimitService.ts`

**功能**:
- `getTodayCountAll()` - 查询今日所有类型使用次数
- `getTodayCountByType()` - 查询今日指定类型使用次数
- `checkUsageLimit()` - 检查是否超过使用限制

**配置常量**:
```typescript
export const USAGE_LIMITS = {
  CHAT: 100,    // 对话次数限制
  IMAGE: 5,     // 上传图片次数限制
  DIARY: 3,     // 生成日记次数限制
};
```

### 2. 集成到业务逻辑 ✅
**文件**: `hooks/useChat.ts`

**修改点**:
1. 导入 `checkUsageLimit` 函数
2. 在 `sendMessage` 函数开始处检查对话次数
3. 在 `uploadImageAndUnderstand` 函数开始处检查上传图片次数
4. 在 `generateDiary` 函数开始处检查生成日记次数

### 3. 检查逻辑
```typescript
// 检查今日对话次数限制
const limitCheck = await checkUsageLimit(user.id, 'chat');
if (!limitCheck.allowed) {
  Alert.alert('提示', limitCheck.message || '您已超过当日的体验次数');
  return;
}
```

## 用户体验

### 提示文案
- 对话超限: "您今日的对话次数已达上限（100次），请明天再试"
- 上传图片超限: "您今日的上传图片次数已达上限（5次），请明天再试"
- 生成日记超限: "您今日的生成日记次数已达上限（3次），请明天再试"

### 交互流程
1. 用户触发操作（发送消息/上传图片/生成日记）
2. 系统检查今日使用次数
3. 如果未超限，正常执行操作
4. 如果已超限，显示提示弹窗，阻止操作

### 容错处理
- 如果检查接口失败，默认允许操作（避免影响用户体验）
- 错误信息会在控制台输出，便于调试

## 技术细节

### 接口调用时机
- **对话**: 在发送消息到大模型之前
- **上传图片**: 在上传图片并调用图片理解之前
- **生成日记**: 在调用日记生成接口之前

### 异步处理
所有检查都是异步的，使用 `await` 等待结果后再决定是否继续操作。

### 状态管理
- 不在客户端缓存使用次数
- 每次操作都实时查询服务端
- 确保数据准确性

## 后续优化建议

### 1. 本地缓存
可以考虑在客户端缓存今日使用次数，减少接口调用：
- 首次操作时查询并缓存
- 每次操作后更新缓存
- 跨天时清空缓存

### 2. 进度提示
可以在界面上显示今日剩余次数：
- "今日剩余对话次数: 85/100"
- "今日剩余上传次数: 3/5"
- "今日剩余日记次数: 2/3"

### 3. VIP 功能
可以为 VIP 用户提供更高的限制：
- VIP 对话次数: 无限制
- VIP 上传图片: 50次/天
- VIP 生成日记: 20次/天

### 4. 优雅降级
当接近限制时提前提示：
- 剩余10次时: "您今日对话次数即将用完"
- 剩余1次时: "这是您今日最后一次对话机会"

## 配置说明

### 修改限制次数
如需修改每日限制次数，编辑 `services/usageLimitService.ts`:

```typescript
export const USAGE_LIMITS = {
  CHAT: 100,    // 修改对话次数限制
  IMAGE: 5,     // 修改上传图片次数限制
  DIARY: 3,     // 修改生成日记次数限制
};
```

### 修改提示文案
提示文案在 `checkUsageLimit` 函数中生成，可以根据需要自定义。

## 注意事项

1. **时区问题**: 确保服务端和客户端使用相同的时区判断"今天"
2. **并发问题**: 如果用户快速连续操作，可能会有并发检查的情况
3. **缓存问题**: 目前不使用缓存，每次都实时查询，确保准确性
4. **用户体验**: 检查失败时默认允许操作，避免影响用户体验
